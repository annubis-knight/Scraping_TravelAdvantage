Implémente la story indiquée dans : $ARGUMENTS

## Workflow

### 1. Comprendre
- Lis la story en entier (problème, solution, critères d'acceptation)
- Note les fichiers listés dans "Fichiers à modifier"

### 2. Analyser l'état actuel
- Lis chaque fichier cible pour voir son état ACTUEL
- Identifie les différences avec ce que décrit la story
- Si le code a évolué, adapte la solution en conséquence

### 3. Implémenter
- Applique les modifications en respectant le style existant
- Si la story propose du code, adapte-le au contexte actuel

### 4. Valider
- Vérifie chaque critère d'acceptation ✓
- Exécute les commandes de test si indiquées

### 5. Finaliser
- Mets à jour le statut à `DONE` dans INDEX.md

## Contraintes
- Ne modifie que les fichiers nécessaires
- Si un conflit existe entre la story et le code actuel, explique-moi avant d'agir
